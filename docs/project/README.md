# Project operational memory

This directory contains lightweight operational memory for the product.

Purpose:

- preserve product context
- maintain strategic continuity
- reduce AI-agent drift
- document important operational decisions
- improve long-term maintainability
- support repeatable delivery workflows

These documents are intentionally:

- lightweight
- operational
- implementation-oriented
- low-maintenance

This is NOT:

- corporate process
- project management bureaucracy
- exhaustive documentation

The goal is:

- stable product cognition
- operational clarity
- repeatable AI-assisted delivery

Core principle:  
The platform exists to accelerate product delivery, not become the product itself.
