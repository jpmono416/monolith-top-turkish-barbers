# Product context

## Product identity

### Name

Top Turkish Barbers Demo Website

### Mission

Demonstrate how modern premium-quality websites can help local independent businesses improve online presence, customer trust and appointment acquisition through low-friction modern digital experiences.

### Target users

- Local independent barber shops
- Walk-in and appointment-based customers
- Mobile-first local customers searching for barber services

### Validation goals

- Validate in-person demo sales workflow
- Validate conversion potential of tailored local-business demos
- Validate rapid AI-assisted product delivery workflow
- Validate premium UX differentiation for small local businesses

---

## Business context

### Monetisation model

- One-time setup/design fee (£300-500)
- Monthly hosting and maintenance subscription (~£30/month)
- Potential future upsells:
  - booking management
  - WhatsApp automation
  - SEO improvements
  - content updates

### Operational constraints

- Extremely fast implementation required
- Minimal operational complexity
- Low-maintenance infrastructure
- No admin panel for initial validation
- No real scheduling engine for MVP

### Success criteria

- Successful in-person demo presentation
- Positive business-owner reaction
- Validation of willingness to pay
- Production-quality perceived polish
- Stable deployment and booking flow demo

---

## Technical context

### Approved stack deviations

- WhatsApp Cloud API developer sandbox integration for prototype/demo validation

### Infrastructure constraints

- Vercel frontend deployment
- Railway API deployment
- Resend email delivery
- Minimal placeholder Supabase wiring
- Minimal placeholder Upstash wiring

### Complexity constraints

- No real booking engine
- No calendar sync
- No authentication
- No admin dashboard
- No queue infrastructure usage initially
- No unnecessary abstractions

---

## Product philosophy

### Priorities

- maintainability over elegance
- operational simplicity
- fast delivery
- low cognitive overhead
- premium visual presentation
- strong first impression
- mobile-first experience

### Explicit non-goals

- enterprise scheduling platform
- multi-business support
- advanced CMS functionality
- internal operational tooling
- generic reusable booking framework

---

## Current implementation stage

Initial local-business validation prototype.

---

## Important context

This product is intentionally designed as a premium-quality but operationally lightweight prototype to validate local-business acquisition workflows and recurring maintenance revenue.

The booking experience is intentionally simplified into an appointment request flow rather than a real scheduling system in order to maximise UX quality while minimising operational complexity.

Visual polish and emotional impact during in-person demos are prioritised over backend sophistication.
