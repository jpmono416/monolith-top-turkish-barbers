# Specifications

Feature specifications and RFCs live here. Specs are the source of truth for spec-driven development.

## Naming

`YYYY-MM-DD-<kebab-feature>.md`

Example: `2026-05-20-order-checkout.md`

Use separate files for product, technical, and plan when complexity warrants it:

- `2026-05-20-orders-product.md`
- `2026-05-20-orders-technical.md`
- `2026-05-20-orders-plan.md`
